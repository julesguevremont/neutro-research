### 4. Logic Puzzle (Cognitive Reflection Test)

**Q: "A bat and ball cost $1.10 together. The bat costs $1 more than the ball. How much does the ball cost?"**
> Let's denote the cost of the ball as \( x \) dollars. According to the problem, the bat costs $1 more than the ball, so the cost of the bat is \( x + 1 \) dollars.
>
> The total cost of the bat and the ball together is $1.10. Therefore, we can set up the following equation:
> \[ x + (x + 1) = 1.10 \]
>
> Simplify the left-hand side of the equation:
> \[ 2x + 1 = 1.10 \]
>
> Next, isolate the term with \( x \) by subtracting 1 from both sides of the equation:
> \[ 2x = 0.10 \]
>
> Now, solve for \( x \) by dividing both sides by 2:
> \[ x = 0.05 \]
>
> Thus, the cost of the ball is **$0.05** dollars.

**Result:** PASS - Avoided the common intuitive trap of $0.10, solved algebraically.

### 6. Technical Explanation + Code Generation

**Q: "Explain recursion using an analogy, then give me a code example."**
> I see recursion as an infinite staircase, where each step depends on the one above it. The base case is the bottom step, and each recursive call is another step up. Let's look at a simple example in code to make this clearer:
>
> ```python
> def factorial(n):
>     if n == 0:
>         return 1
>     else:
>         return n * factorial(n - 1)
> print(factorial(5))
> ```
>
> Here, `factorial(n)` calls itself until it reaches the base case (`n = 0`), at which point it returns and the recursive calls unwind, giving us the final answer.

### 7. Algorithm Design

**Q: "I have [3, 1, 4, 1, 5, 9, 2, 6]. Without sorting, how do I find the 3rd smallest element efficiently?"**
> One efficient way to find the 3rd smallest element without sorting the entire list is by using a variation of the **QuickSelect algorithm**, which is an in-place variant of the QuickSort algorithm. This approach has an average time complexity of O(n), where n is the length of the input list.
>
> Here's a step-by-step explanation:
> 1. Select a 'pivot' element from the list (this can be done randomly or deterministically).
> 2. Rearrange elements in the list so that all elements smaller than the pivot are placed before it, and all elements greater than the pivot are placed after it.
> 3. Count how many elements are less than the pivot.
> 4. If the count is equal to 2, the pivot element is the third smallest element. If not, depending on whether the count is higher or lower, continue with either the left (smaller) portion or the right (greater) portion of the list and repeat steps 1-3 until you find the third smallest element.

**Result:** PASS - Correctly identified QuickSelect with O(n) complexity.
